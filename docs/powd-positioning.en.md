# `powd` Positioning

## Position

`powd` should not be positioned as a standalone miner or a token product.

The more accurate definition is:

> a product that turns idle local compute into budget an agent can use.

The core point is not "come mine". The core point is that, with user consent, an agent can turn spare compute into a sustainable budget path. That path may later map to AI credits, but that is not the primary story right now.

## Problem it solves

### 1. Lower the cost barrier for agent usage

Many users are cost-sensitive, not agent-averse.

`powd` offers:

- use idle compute to offset agent budget

That is easier to accept than leading with mining or subscription pricing.

### 2. Build a loop for agent products

Traditional flow:

- user pays
- user consumes

This line aims for:

- agent earns budget
- agent spends budget
- user keeps using the agent

That creates an acquisition, retention, and consumption loop.

### 3. Give Starcoin a stronger distribution story

This is not only about a pool and not only about a chain.

It places Starcoin inside a broader story:

- idle compute monetization
- agent runtime
- sustainable budget source

## Target users

Best early users:

- frequent agent users
- local Claw/runtime users
- cost-sensitive users willing to run light background work
- developers, indie hackers, crypto-native AI users

Not the initial focus:

- pure miners
- users who only care about token price
- professional ASIC operators

## What it is

- an activation path for agent products
- a feature suitable for invite-only and growth experiments
- an engine inside the Claw/runtime ecosystem
- a conversion layer from idle compute to usable budget

## What it is not

- a standalone mining rig product
- a primary profit product
- a token issuance project
- a business that depends on CPU mining alone being highly economic

## Growth logic

Suggested order:

1. sell agent usage value first
   - "let your agent earn part of its own budget"
1. introduce Starcoin and mining second
   - mining is the mechanism, not the home-page hero
1. discuss subsidy and token economics last
   - those are optimization layers, not the first-screen pitch
